# AGENTS.md — Kitchen Ecommerce (Laravel 13)

## Quick start
```bash
composer setup          # full setup: install, .env, key, migrate, npm, build
composer dev            # dev server + queue worker + logs + Vite (concurrently)
composer test           # config:clear && artisan test
php artisan test        # run PHPUnit (SQLite :memory:)
npm run build / dev     # Vite (Tailwind + Alpine.js inputs)
php artisan migrate     # SQLite migrations
php artisan db:seed     # seeds admin user + categories + brands + products from CSV
```

## Architecture
- **Framework**: Laravel 13.x, PHP ^8.3, SQLite (session/cache/queue all use `database` driver)
- **Frontend**: Vite + Tailwind CSS v3 + Alpine.js + Bootstrap 5. Entry: `resources/css/app.css`, `resources/js/app.js`
- **Auth**: Laravel Breeze. Routes in `routes/auth.php` and `routes/web.php`
- **Admin**: `AdminMiddleware` checks `auth()->user()->role === 'admin'`. Admin controllers in `app/Http/Controllers/Admin/`
- **Deployment**: Docker (php:8.3-apache) deploys to **Render** at `https://web-e-commerce-online.onrender.com`
- **CI**: GitHub Actions (`laravel.yml`) — push/PR to `main` → PHP 8.3, SQLite, migrate, route:list, test

## Key commands
| Command | Purpose |
|---|---|
| `php artisan db:seed` | Seeds admin (`admin@gmail.com`/`password`) + user (`John@gmail.com`/`password`) + 5 categories + 5 brands + CSV products |
| `php artisan test` | Runs PHPUnit (in-memory SQLite, no `.env` copy needed) |
| `composer pint` | Laravel Pint (PSR-12 code style) |
| `npm install --ignore-scripts` | `.npmrc` sets `ignore-scripts=true` — pass `--ignore-scripts` or override |

## Domain structure
| Directory | Purpose |
|---|---|
| `app/Models/` | Product, Category, Brand, Cart, User (no Order model — see bugs) |
| `database/imports/products.csv` | Product seed data (generated by `amazon-data-cleaning/clean_data.py`) |
| `amazon-data-cleaning/` | Python script that transforms Amazon raw CSV → cleaned `products.csv` for seeding |
| `app/Http/Controllers/Admin/` | Admin CRUD: ProductController, CategoryController |

## Known bugs (fix before editing)
1. **Missing `Order` model** — `app/Models/Order.php` contains an `OrderItem` class (copy-paste error). A proper `Order` model class does not exist. `app/Models/OrderItem.php` also has `OrderItem`.
2. **ProductController::create() bug** — variables named `$category`/`$brand` but `compact()` uses `categories`/`brands`. Will throw undefined variable error.

## Testing quirks
- Tests use `DB_CONNECTION=sqlite` with `:memory:` (see `phpunit.xml`)
- No `RefreshDatabase` trait in ExampleTest — be aware for new tests
- Seeder uses `WithoutModelEvents` trait
